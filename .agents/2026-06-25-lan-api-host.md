# 2026-06-25 局域网访问 API 地址

## 背景

用户反馈同一网络的其他设备打不开或无法正常使用前端。检查发现前端服务和后端服务都监听 `0.0.0.0`，本机通过 `192.168.43.4` 可以访问，但前端 API 默认仍指向 `localhost:8000`。

## 修改

- 修改 `前端/src/api/http.ts`。
- 当前端没有显式设置 `VITE_API_BASE_URL` 时，默认使用当前访问页面的 hostname 拼接 `:8000`。
- 例如别人访问 `http://192.168.43.4:5174` 时，API 会自动请求 `http://192.168.43.4:8000`。

## 验证

- 已确认本机 `http://192.168.43.4:5174` 返回 200。
- 已确认本机 `http://192.168.43.4:8000/health` 返回正常。
- 已运行前端 TypeScript 构建检查：`tsc -b` 通过。
- 尝试添加 Windows 防火墙规则失败，原因是当前 PowerShell 不是管理员权限。
